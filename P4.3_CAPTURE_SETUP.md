# P4.3 — Captura que convierte (Newsletter + Quiz + Wishlist)

Este paquete añade 3 piezas “conversión-first”:

1. **Newsletter** (double opt-in)
2. **Quiz (Travel Matcher)** → recomienda tours + opcionalmente guarda un lead en CRM
3. **Wishlist** (favoritos) → requiere login (Supabase magic link)

---

## 1) Base de datos (Supabase)

Ejecuta el patch SQL:

- `supabase_patch_p43_capture.sql`

Incluye:

- `public.newsletter_subscriptions`
- `public.wishlists`
- `public.wishlist_items`
- RLS + políticas (wishlist por owner; newsletter admin-only)

> Nota: newsletter se escribe vía API server (service-role). No se expone lectura pública.

---

## 2) Variables de entorno

Ya existían en el proyecto y se reutilizan:

- `RESEND_API_KEY`
- `EMAIL_FROM`
- `EMAIL_REPLY_TO` (opcional)
- `NEXT_PUBLIC_SITE_URL` o `SITE_URL`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

---

## 3) Supabase Auth (Magic Link)

En Supabase Dashboard:

- Auth → URL Configuration:
  - **Site URL**: tu dominio (p.ej. `https://kce.co`)
  - **Redirect URLs**: agrega `https://kce.co/auth/callback` (y tu localhost)

El flujo usa:

- `/login` → `signInWithOtp`
- `/auth/callback` → `exchangeCodeForSession` → redirige a `/wishlist`

---

## 4) Rutas y páginas nuevas

### Newsletter

- `POST /api/newsletter/subscribe`
- `GET  /api/newsletter/confirm?token=...`
- `GET  /api/newsletter/unsubscribe?token=...`
- Página: `/newsletter`

### Quiz

- `POST /api/quiz/submit`
- Página: `/quiz`

### Wishlist

- `GET  /api/wishlist` (Bearer access_token)
- `POST /api/wishlist/toggle` (Bearer access_token)
- Página: `/wishlist`
