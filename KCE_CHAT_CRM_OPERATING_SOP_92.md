# KCE Chat + CRM Operating SOP (92)

## Objetivo
Hacer que el chat público funcione como una entrada comercial real, no como una caja de texto aislada.

## Flujo recomendado
1. Detectar intención: catálogo, plan personalizado, booking o soporte.
2. Resumir visualmente la respuesta con bloques cortos.
3. Capturar email/WhatsApp solo con consentimiento.
4. Cuando haga falta, continuar a Contacto con el contexto ya resumido.
5. Si el caso es sensible o listo para seguimiento, abrir lead/ticket/task en CRM.

## Qué debe hacer el equipo
- No reiniciar la conversación si ya viene desde chat o plan personalizado.
- Usar el contexto recibido en Contacto como base de la respuesta.
- Priorizar el siguiente paso útil: tour, plan o humano.
- Escalar soporte/pago/cambio a ticket cuando corresponda.

## Qué no debe pasar
- Pedir dos veces los mismos datos.
- Enviar muros de texto desde el chat.
- Mandar al usuario a Contacto sin ningún resumen del caso.
- Abrir follow-up sin consentimiento o sin señal clara del viaje.
