# P46 — Ops Metrics + Dependency Healthchecks + Auto-recovery cleanup

## Qué entra

1) **Ops metrics / SLA**
- Endpoint admin: `/api/admin/ops/metrics?hours=24|72|168`.
- UI: `/admin/ops/metrics` con KPIs rápidos:
  - Incidentes (total) en ventana.
  - SLA promedio de *ack* y *resolve* (desde `first_seen_at`).
  - Top `kind`.
  - Pausas activas (desde `crm_channel_pauses`).

2) **Healthchecks de dependencias**
- `/api/health` (shallow) + housekeeping: ahora también prunea pausas expiradas.
- `/api/health/deps`:
  - `deep=0` (default): solo env presence + supabase ping.
  - `deep=1`: llama con timeout a Stripe/Resend (rápido) para confirmar conectividad.

3) **Auto-recovery housekeeping (pausas expiradas)**
- Nueva utilidad `pruneExpiredChannelPauses()` en `src/lib/channelPause.server.ts`.
- Se ejecuta en `/api/health` para mantener `crm_channel_pauses` limpio.

## SQL
Ejecutar (una sola vez) en Supabase:
- `supabase_patch_p46_ops_metrics.sql`

## Notas
- Los checks deep de Stripe/Resend usan timeout 2.5s y devuelven `skipped=true` si falta la API key.
- Las métricas agregan en Node para evitar RPC/SQL complejos y mantenerlo portable.
