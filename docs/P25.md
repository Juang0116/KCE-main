# P25 — Auto-mitigation (CRM Autopilot)

## Objetivo
Reducir incidentes operativos (spikes de fallos, caídas de paid-rate) con:
- Alertas server-side basadas en métricas reales.
- Acciones automáticas conservadoras (pausas temporales, disable auto-weights, ticket interno).
- Ledger auditable para Revenue Ops.

## Qué se agregó
### SQL
- `supabase_patch_p56_auto_mitigation.sql`
  - `crm_runtime_flags`
  - `crm_channel_pauses`
  - `crm_alert_rules`
  - `crm_alerts`
  - `crm_mitigation_actions`
  - Seeds idempotentes de reglas base:
    - `failed_rate_spike`
    - `paid_rate_drop`

### Server
- `src/lib/runtimeFlags.server.ts` — feature flags via DB (cache 60s).
- `src/lib/channelPause.server.ts` — pausas por canal.
- `src/lib/alerting.server.ts` — evaluación de alertas.
- `src/lib/mitigations.server.ts` — acciones automáticas + ledger + ticket interno.
- Cron autopilot ahora evalúa alertas + corre mitigaciones (best-effort).

### API
- `GET /api/admin/metrics/alerts?days=14`
- `GET /api/admin/metrics/alerts?days=14&run=1&dryRun=true` (evaluar on-demand)
- `GET /api/admin/metrics/alerts/cron` (Bearer CRON_SECRET)
- `GET /api/admin/metrics/mitigations?days=14`

### UI
- `/admin/metrics` ahora muestra:
  - Últimas alertas
  - Acciones aplicadas (ledger)
  - Botón “Evaluar ahora (dry-run)”

## Guardrails
- Si `crm_channel_pauses` tiene `email` pausado, `processOutboundQueue` cancela envíos (status `canceled`) y deja traza `outbound.paused`.
- `templateOptimization.server.ts` respeta `crm_runtime_flags.crm_auto_promote_weights=false` (override de env) para no tocar pesos.

## Flags
- DB: `crm_runtime_flags`
  - `crm_auto_promote_weights = false|true` (override de `CRM_AUTO_PROMOTE_WEIGHTS`)
