# Fase 6.1 — Conversion polish

Este avance refuerza la capa comercial sin tocar la lógica crítica de pagos:

- Ficha de tour con señales de confianza más visibles.
- Widget de reserva con prueba social funcional (Stripe, invoice, soporte).
- Post-pago más claro en checkout/success y booking.

## Qué validar

1. Abrir un tour y revisar el sidebar en desktop/mobile.
2. Confirmar que el widget sigue enviando a Stripe sin errores.
3. Completar un pago de prueba y revisar las nuevas tarjetas de success/booking.
4. Confirmar que links de invoice/calendar/support siguen operando.
