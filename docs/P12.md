# P12 — Closed-loop replies + no-reply follow-ups

## What’s new
1) **Auto-mark replied**
- **Web chat**: when the customer sends a message (role=`user`) in a conversation, the system tries to find the latest `crm_outbound_messages` for the active deal and marks it as `outcome='replied'`.
- **Email inbound webhook**: endpoint `/api/webhooks/resend/inbound` can be called by Resend inbound webhooks. It matches the sender email with the latest outbound email sent (within 14 days) and marks it `replied`.

2) **No-reply follow-ups (24h/48h)**
- The cron trigger `runSalesOutboundTriggers` now checks active deals in stages `proposal` and `checkout`. If the last outbound stage message was sent but no reply was detected, it will enqueue a follow-up at 24h and then another at 48h (once each).

3) **Checkout link persistence**
- Deals can store `checkout_url` and `stripe_session_id` so follow-ups can reuse the exact payment link.

## Required Supabase patch
Run:
- `supabase_patch_p12_deals_checkout_url.sql`

## Env vars
Add to `.env.local`:
- `RESEND_INBOUND_TOKEN` (protect inbound webhook)
- `CRM_FOLLOWUP_NOREPLY_24H` and `CRM_FOLLOWUP_NOREPLY_48H` (optional)

## Resend inbound webhook config
- Configure the webhook URL to `https://<your-domain>/api/webhooks/resend/inbound`
- Send `Authorization: Bearer <RESEND_INBOUND_TOKEN>`
