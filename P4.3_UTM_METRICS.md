# P4.3 — UTM Attribution & Conversion Metrics

## Goal

Provide lightweight attribution (UTM cookies) and a decision-ready view of conversion by campaign.

## Local Development (port 3000)

Use port **3000** for the Next.js app.

Example link to test UTM capture:

- `http://localhost:3000/tours?utm_source=ig&utm_medium=social&utm_campaign=jan8`

## What is captured

- Cookie `kce_vid`: random visitor id (30 days).
- Cookie `kce_utm`: JSON (30 days) with:
  - `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`, `utm_content`
  - `gclid`, `fbclid` when present

## Events written (table: `public.events`)

- `marketing.utm_capture`
- `newsletter.signup_confirmed`
- `quiz.completed`
- `checkout.paid`
- `wishlist.toggled`

Each conversion event includes `vid` and `utm_*` in the `payload`.

## Admin report

Open: `/admin/metrics` → section **UTM / Campañas**.

The report shows counts and rates:

- Quiz/Capture
- Confirm/Capture
- Paid/Capture
- Paid/Quiz

Window defaults to last 7 days unless you set `from`/`to`.

## API

`GET /api/admin/metrics/utm?from=YYYY-MM-DD&to=YYYY-MM-DD` Returns `{ summary, items[] }`.
